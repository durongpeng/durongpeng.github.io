    Spring的三种装载bean的方式

    创建时间:2019-03-10 12:33

    • Spring容器负责创建应用程序中的bean并通过DI来协调这些对象之间的关系。Spring提供了三种方式来让开发人员告诉Spring要创建哪些bean，并且如何将其装配到一起。
    开始之前先定义两个类来进行测试：一个简单的User类(setter和getter方法就忽略了)，一个注入User的UserServiceImpl类。

    public class User {    
    private String name;    
    private String pwd;    
    private int age;    
    private User(){        
        System.out.println("无参方法"); 
    }    
    private User(String n,String p,int a){       
        this.name = n;        
        this.pwd = p;        
        this.age = a;       
        System.out.println("有参方法");    
    }
}

public class UserServiceImpl  {    
    private User user;    
    public User getUser() {        
        return user;   
    }   
    @Autowired    
    public void setUser(User user) {        
        this.user = user;    
    }
}

    基于XML的bean定义

    在XML中要以元素作为根标签，使用标签来声明一个bean。id属性定义bean的id，class属性定义bean的全限定类名。如果没有明确给出id，那么Spring会给bean生成一个id，大约的格式是包名+类名+”#”+数字(该bean的第几个对象)

        <!-- 注册userService -->
    <bean id="userService" class="top.durp.service.impl.UserServiceImpl"/>

    <!-- 注册bean -->
    <bean id="user1" class="top.durp.model.User">
        <property name="age" value="1"/>
        <property name="pwd" value="pwd"/>
        <property name="name" value="name"></property>
    </bean>

    标签下有九个属性：

    id和name属性

    id是bean的唯一标签，name是bean的名字。两者可以一致，两者都不能与其他bean的重复。若bean的name和另一个bean的id相同，在创建bean时会报错。通常使用id属性，name属性基本不用。
    name属性可以指定多个名字，名字之间可以用逗号、分号或者空格分开。

    <bean id="user2" name = "user2;user4" class="top.durp.model.User">
    class属性

    bean的全限定类名，指向classpath下类定义所在位置。

    factory-bean和factory-method

    通过factory-bean属性我们可以调用一个指定bean来作为工厂bean。
    通过factory-method属性我们可以调用一个指定的静态工厂方法来创建bean的实例。

    public class UserFactory {    
    public static User createUser(){        
        return  new User();    
    }
}

    <bean id="userFactory"  class="top.durp.test.UserFactory" factory-method="createUser"/>
#factory-bean和factory-method搭配使用
<bean id="user5"   factory-bean="userFactory" factory-method="createUser" />
    init-method和destory-method属性

    init-method属性是bean的初始方法，在创建好bean后调用该方法。
    destory-method属性是bean的销毁方法，在销毁bean之前调用该方法，一般在该方法中释放资源。

    scope属性

    scope属性有四个值：

    • singleton：表示整个IOC容器共享一个bean，每次通过getBean()获取的bean都是同一个。
    • prototype：每次对该bean请求(将其注入另一个bean或通过getBean()方法获取)时都会创建一个新的bean实例。
    • request：每次HTTP请求将会生成各自的bean实例。
    • session：每次会话请求对应一个bean实例。
    autowire属性

    autowire属性表示bean的自动装配，有六个值：

    • no：默认值，不进行自动装配。
    • byName：根据属性名自动装配。此选项将检查容器并根据名字查找与属性完全一致的bean，并将其与属性自动装配。
    • byType：如果容器中村子啊一个月指定属性类型相同的bean，那么将于该属性自动装配；如果存在多个该类型的bean，则抛出异常；如果没有找到类型相匹配的bean，则什么都不会发生，可以通过dependency-check=”objects”让Spring抛出异常。
    • constructor：与byType方式类似，不同之处在于它应用于构造器参数。如果容器中没有找到构造器参数类型一致的bean，那么抛出异常。
    • autodetect：通过bean类的内省机制(introspection)来决定是使用constructor还是byType方式进行自动装配。如果发现默认的构造器，那么将使用byType方式，否则使用constructor。

    • default：由上级标签的default-autowire属性确定。

      depends-on

      表示一个Bean的实例化依靠另一个Bean先实例化

    基于注解的bean定义

    Spring提供了四个注解，这四个注解与XML中定义bean的标签效果一致，将组件交给Spring进行管理。

    • @Component：当对组件的层次难以定位的时候使用这个注解。
    • @Controller：表示控制层组件。
    • @Service：表示业务逻辑层的组件。
    • @Repository：表示数据访问层的组件。
      开启组件扫描
      使用注解需要注意在XML配置文件中开启注解扫描或者使用@ComponentScan注解开启组件扫描。
    • XML中开启组件扫描
      <context:component-scan base-package="top.durp.model">
    • @ComponentScan开启组件扫描
      //如果不指定组件名字，则默认是类名的小写
@Component("user")
@ComponentScan(basePackages = {"top.durp.model","top.durp.service"})
public class User {   
  private String name;    
  private String pwd;    
  private int AGE;    
  public User(){        
  System.out.println("无参方法");    
}
      @ComponentScan不设置任何属性，它会以类所在的包作为基础包来扫描组件，如果想设置多个基础包，可以使用basePackages属性。

    基于Java代码装配bean

    在大多数场景下，使用@Autowired和@Component来自动装配bean是更为推荐的方式，但有些情况此方案是行不通的，例如需要将第三方类库中的组件装配到你的应用中，在这种情况下是无法再jar中的类上加注解的，因此就无法使用自动化装配的方案。
    在进行显式配置时，使用JavaConfig是更好的方案，它更强大、类型安全并且对重构友好。因为它就是Java代码，就像应用程序中的其他Java代码一样。同时，JavaConfig又与其他Java代码有所区别，JavaConfig是配置代码，这意味着它不应该包含业务逻辑，也不应该被加入到业务逻辑中去。尽管不是必须的，将JavaConfig放在单独的包中，与其他业务逻辑程序分离开，能更清晰的提现其在程序中的配置作用。

    @Configuration
@ComponentScan(basePackages = {"top.durp.model","top.durp.service"})
public class JavaConfig {
    @Bean(name = "user1")
    public User getUser(){    
        return new User();
    }
    @Beanpublic UserService getUserServiceImpl(){    
        return new UserServiceImpl();
    }
}

    @Bean注解会告诉Spring这个方法将会返回一个对象，该对象要注册为Spring应用上下文中的bean。方法体中包含了最终产生bean实例的逻辑。默认情况下bean的id与带有@Bean注解的方法名是一样的。

    Spring三种常用的注入方式

    Java Java Spring
    创建时间:2019-03-06 09:48
    阅读:
    1. 构造器方法注入
    2. setter方法注入
    3. 基于注解的注入
      1. @Autowired
      2. @Resource

    构造器方法注入

    由构造方法注入通常是在Spring的配置文件中，用标签声明id和class属性，然后用若干个标签来判断使用该类的哪一个构造器。Spring在创建bean时会根据标签配置来判断哪一个构造器，如果没有符合的构造器，则会抛出异常。
    先给出一个简单的User类(setter和getter方法就忽略了)

    public class User {    
    private String name;    
    private String pwd;    
    private int age;    
    private User(){        
        System.out.println("无参方法"); 
    }    
    private User(String n,String p,int a){       
        this.name = n;        
        this.pwd = p;        
        this.age = a;       
        System.out.println("有参方法");    
    }
}

    下面贴出Spring配置文件中bean的声明，会发现Spring能够按照给出的参数创建bean，但若是将第三个值修改成字母，则创建失败，因此说明在标签在不指定index、name、type属性时，是按照参数列表从左到右进行装填的，选择的构造函数仅仅是参数个数相同，若有两个相同参数的方法，则会根据参数类型进行判断。

    <bean id="user" class="top.durp.model.User">          
    <constructor-arg value="name"></constructor-arg>       
    <constructor-arg value="pwd"></constructor-arg>       
    <constructor-arg value="1"></constructor-arg>
</bean>

    标签有五个属性:

    • type：用来确定参数的类型
    • name：用来确定参数在形参列表中的名字
    • index：用来确定参数在形参列表中的序号
    • value：传入值
    • ref：传入其他bean对象
      通过下面这个例子你会发现，标签没有根据方法形参列表的顺序来定义，而是随意定义的，但保证了根据定义能找到合适的构造方法。
      <bean id="user" class="top.durp.model.User">      
  <constructor-arg type="int" value="1"></constructor-arg>      
  <constructor-arg name="n" value="name"></constructor-arg>      
  <constructor-arg index="1" value="pwd"></constructor-arg>
</bean>

      setter方法注入

      setter方法注入很简单，用标签指定属性名称和属性值，即可进行属性注入。但有一个重要的前提，那就是类中必须为属性提供setter方法，而且是public的方法，否则创建bean时会报错。 
      <bean id="user" class="top.durp.model.User">
  <property name="age" value="1"/>
  <property name="pwd" value="pwd"/> 
  <property name="name" value="name"></property>
</bean>
      需要特别注意的是name属性的值和bean的属性名并没有直接关系，只与提供的setter方法的方法名有关。Spring会将name值的首字母转换成大写，然后再拼上”set”构成setter方法名，通过反射调用，实现注入。下面的代码也是不会报错的。
      private int AGE;
public void setAge(int a){
  this.AGE = a;
}
<property name="age" value="1"/>

    基于注解的注入

    @Resource和@Autowired都是做bean的注入时使用，其实@Resource并不是Spring的注解，它的包是javax.annotation.Resource，需要导入，但是Spring支持该注解的注入。
    为了试验基于注解的注入，引入一个新类

    public class UserServiceImpl  {    
    private User user;    
    public User getUser() {        
        return user;   
    }   
    @Autowired    
    public void setUser(User user) {        
        this.user = user;    
    }
}

    修改Spring配置文件

    <bean id="userService" class="top.durp.service.impl.UserServiceImpl"/>

    @Resource和@Autowired既可以用在属性上，也可以用在setter方法上，因此如果都用在属性上，就可以省略setter方法了

    @Autowired

    @Autowired还可以用在构造器上,用在构造器上和用在属性上的区别是加载顺序不同。用在构造器上，构造的时候就要准备好User对象，因此先创建User的bean。用在属性上，是先创建好UserServiceImpl的的bean，再注入User的bean，因此User的bean创建时间上晚于UserServiceImpl的bean。

    @Autowired
public UserServiceImpl(User u){
    this.user = u;
}

    @Autowired默认是ByType来自动装填的，当一个类型有多个bean存在于Spring中时，自动装填就会抛出异常，这时候需要@Qualifier注解来指定一个id，装填此id的bean。可以通过required属性来控制是否必须装填，默认为true。

    @Autowired()
@Qualifier("user")
private User user;
    @Resource

    @Resource默认按照ByName自动注入，不怕有多个相同类型的bean。只要Spring中有一个bean的id和注入属性值一样，就可以注入进去。但如果用@Resource的name属性指定了注入bean的名称，则要保证Spring容器中bean的id与指定的名称要一致。

    @Resource(type = top.durp.model.User.class,
name = "user1")
private User user;

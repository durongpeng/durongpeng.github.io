    串

    由零个或多个字符组成的有限序列，又名叫字符串。

    串的比较

    两个数字比较大小大家都会，但是两个字符串比较是啥意思？汉字还分大小？这就要先普及一下ASCII和Unicode了。

    ASCII和Unicode

    ASCII编码是美国人发明的，由7位二进制数表示一个字符，总共可以表示128个字符。后来发现字符种类太多表示不过来了，就把ASCII扩至8个二进制位，能表示256个字符。但这仅仅是够美国人表示自己的符号。为了表示实际上成百上千种语言的符号，就有了Unicode编码。
    Unicode编码比较常用的是由16位二进制数表示一个字符，大约是6.5万多个字符。为了和ASCII码兼容，Unicode的前256个字符与ASCII编码完全相同。

    比较字符串的步骤

    假设给定两字符串s=”a1a2a3······an”,t=”b1b2b3······bn”

    1. 先比较两串长度大小，长度大的串>长度小的串
    2. 如果两串的长度相等，从头循环比较两字符串的每一位字符，比较标准就是Unicode编码了，直到比较出在某一处，s的字符对应的Unicode值不等于t的字符，返回比较结果。

      串的存储结构

      串的顺序存储结构

      串的顺序存储结构是用一组地址连续的存储单元来存储串中的字符序列的。按照预定义的大小，为每个定义的串变量分配一个固定长度的存储区。一般是用定长数组来定义。
      一般会将串的实际长度存放在数组0下标处或最后一个下标处，也有串的后面跟”\0”表示串的结束。串的实际长度并不记录，而是通过遍历来计算得到。

      串的链式存储结构

      串的链式存储与线性表是相似的，但由于串中的每个数据元素都是一个字符，如果一个节点对应一个字符就太浪费空间了。因此一个节点可以存放多个字符，最后一个节点未被占满时，用”#”或其他非串值字符补全。
      一个节点存放多少个字符合适就很重要，这会影响串处理的效率，需要根据实际情况做出选择。

      朴素的模式匹配算法

      对主串S的每一个字符作为子串开头，与要匹配的字符串T进行匹配。对主串S做大循环，每个字符开头做子串T长度的小循环，直到匹配成功或全部遍历完成为止。
      /* 返回子串T在主串S中第pos个字符之后的位置。若不存在则返回0 */
/* T非空，1≤pos≤strLength(S) */
int Index(String S,String T,int pos) 
{
 int i = pos ;        //主串中的下标
 int j = 1 ;          //子串中的下标
 while ( i ≤ S[ 0 ] && j <= T[ 0 ] ) {  // i小于S串长度&&j小于T串长度
     if ( S[ i ] == T[ j ] ) {     // 两字符相等则继续向下比较
         ++i ;
         ++j ;
     } else {                  // 两字符不等，指针后退重新比较
         i = i - j +2 ;      // i退回到上次匹配首位的下一位
         j = 1；            // 子串下标回到首位
     }
 }
     //当j增长到大于T串长度，说明已经在S主串中匹配到T子串，返回S主串处的索引值
     if ( j > T [0] ) {
         return i - T [0]; 
     } 
     else return 0;
}

      朴素模式匹配算法的时间复杂度

      设主串S，长度n;子串T，长度m。
      最好的情况就是主串第一位就和子串第一位匹配，并且后面的也都匹配，时间复杂度为O[1]。(书上写的是O[1],个人理解为O[m])
      再差一点的情况就是,在主串和子串的比较中，对比子串的第一位时就能发现不匹配，而不是对比一段之后才会发现不匹配。这种情况只需要对比子串的开头，而不需要对比子串后面部分的元素，因此时间复杂度是O[n+m]。(最好情况:第一位就开始匹配，时间复杂度O[m],每一次都不匹配，时间复杂度O[n]，平均计算(m+n)/2)
      最差的情况就是，主串和子串的每次对比，都是对比到子串的最后一个元素时才发现不匹配，这种情况的时间复杂度为O[(n-m+1)*m]

      KMP模式匹配算法

      KMP模式匹配算法原理

      假设在朴素的模式匹配算法，匹配主串S=”abcdefgab”和子串T=”abcdex”，前五个字母完全相同，直到第六个字母”f”和”x”开始不相同，朴素算法示的意图如下 对于要匹配的子串T来说，”abcdex”首字母”a”与后面的串”bcdex”中任意一个字符都不相等。也就是说，既然”a”不与自己后面的子串中任意一个字符相等，那么上图①来说，前五位字符分别相等，意味着子串T的首字符”a”不可能与S串的第2到5位的字符相等。在上图中②③④⑤步的判断都是多余的。
      理解KMP算法的关键在于你能确定T串中”a”与后面的字符均不相等(这是前提，至于如何判断后面再讨论)。 可以推出上面朴素算法中的②③④⑤都可以去掉，只保留①⑥即可。
      或许有人觉得要满足这个前提限制性太大，那么再来看一下如果去掉这个限制会怎么样。假设有主串S=”abcababca”和子串T=”abcabx”。对于初次判断，两串前五个字符相等，第6个字符开始不等，如步骤①。根据①的比较,T的首字符”a”与T的第二位字符”b”、第三位字符”c”均不相等，所以不需要做判断，因此步骤②③是多余的。 因为T的首位”a”与T的第四位”a”相等，第二位”b”与第五位”b”相等。而在①时，第四位的”a’与第五位的”b”已经与主串S上的四五位比较过是相等的，因此可以断定T串的四五位与S串的四五位也是相等的(T[4]=S[4]&T[1]=T[4]→T[1]=S[4])。所以④⑤步也是多余的。
      对比这两个例子发现，我们发现在①时，主串索引下标i的值为6，②③④⑤四步的i值分别是2、3、4、5，到了⑥，i值又变回了6。即我们在朴素模式匹配算法中，主串的i值是不断地回溯来完成的。KMP算法就是为了让没必要的回溯不发生。既然i不回溯（不会变小），我们就要考虑j值的变化了。对比上面两图，我们发现，j的变化与T串的首字符和自身后面字符是否相等有关，而与主串无关，即j值的变化关键取决于T串的结构中是否有重复的问题。
      如一图中，T=“abcdex”，当中没有任何重复字符，所以j由6变为1，二图中，T=“abcabx”，前缀“ab”与最后“x”前的“ab”相等，因此j就由6变成了3。因此可以得出规律，j值的多少取决于当前字符之间的串的前后缀的相似度。我们把T串各位置的j值的变化定义为一个数组next，那么next的长度就是T串的长度，则有如下函数定义：

      next数组值推导

      T=”abcabx”
    • j = 1 时, next[1] = 0
    • j = 2 时, j 由 1 到 j-1 只有字符”a”，属于其他情况 next[2] = 1
    • j = 3 时, j 由 1 到 j-1 串是”ab”，”a”和”b”不相等，属于其他情况 nextt[3] = 1
    • j = 4 时, j 由 1 到 j-1 串是”abc”，三个字符各不相等，属于其他情况 nextt[4] = 1
    • j = 5 时, j 由 1 到 j-1 串是”abca”，前缀字符”a”和后缀字符”a”相等，通过上面的公式算出k值为2，因此next[5] = 2
    • j = 6 时, j 由 1 到 j-1 串是”abcab”，前缀串”ab”和后缀串”ab”相等，因此next[6] = 3
      根据经验得出，如果前后缀一个字符相等，k值为2，两个字符相等，k值为3，n个字符相等，k值为n+1。

      KMP模式匹配算法实现

      /* 通过计算返回子串T的next数组 */
void get_Next(String T, int *next){
 int i,j;
 i = 1;  //next数组和T串元素索引，从1开始
 j = 0;  
 next[ 1 ] = 0;        // j = 1 时 next[ j ] = 0
 while(i < T[ 0 ]){
     if( j == 0 | | T[ i ] == T[ j ] ){
         ++i;
         ++j;
         next[ i ] = j;
     } else {
         j = next[ j ];   // 字符不相同，j 值回溯
     }
}
/* 返回子串T在主串S中第pos个字符之后的位置。若不存在，则函数返回值为0. */
index index_KMP(String S, String T, int pos){
 /* 主串当前位置下标值，若pos不为1则从pos位置开始匹配 */
 int i = pos;
 /* 用于子串T中当前位置下标值 */
 int j = 1;
 /* 定义next数组 */
 int next[ 255 ];
 / * 对T子串分析得到其next数组 */
 get_next( T,next );
 /* i小于主串S长度&j小于子串T长度时循环 */
 while ( i <= S[ 0 ] & & j<=T[ 0 ] ){
     /* 两字母相等时则继续，相较于朴素模式匹配算法多了j==0的判断 */
     if ( j == 0 | | S[ i ] == T[ j ] ){
         ++i;
         ++j;
     } else {                         // 指针后退重新开始匹配
         j = next[ j ];              // j 退到合适的位置，i值不变
     }
 }
 if ( j > T[ 0 ] ) {
     return i - T[ 0 ]
 } else {
     return 0;
 } 
}

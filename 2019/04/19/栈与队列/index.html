    栈

    栈(Stack)是限定仅在表尾进行插入和删除操作的线性表。允许插入和删除的一端称为栈顶，另一端称为栈底。栈又称为后进先出的线性表，简称LIFO结构。

    栈的顺序存储结构

    栈是线性表的特例，栈的顺序存储其实就是线性表顺序存储的简化，简称顺序栈。用下标为0的一端作为栈底比较好，因为首元素都存在栈底，变化最小，所以让它作为栈底。

    栈的结构定义

    typedef int ItemType   
typedef strut 
{
    ItemType data[MAXSIZE];
    int top；
}MyStack

    定义一个int类型的top变量来指示栈顶元素在数组中的位置，若栈的长度为StackSize，则top必须要小于StackSize，当栈中存在一个元素时，top等于0，空栈的top值为-1。

    顺序栈的存取操作都没有循环，因此其时间复杂度为O[1]。

    进栈

    Status Push(MyStack *S,ItemType e){
    if(S->top == MAXSIZE-1) {
        return ERROR;                                 //栈满了
    }
    S->top++;                               //top值加1
    S->data[S->top] = e;                //新元素赋值给栈顶空间
    return ok;
}

    出栈

    Status Pop(MyStack *S,ItemType *e){
    if(S->top == -1) {
        return ERROR;                                 //空栈
    }
    return ok;
    *e = S->data[S->top] ;                //出栈的元素赋值给e
    S->top--                              //top值减1
    return ok；
}

    两栈共享空间

    两个栈有两个栈底，让一个栈的栈底为数组的始端，即下表为0处，另一个栈的栈底为数组的末端，即下表为数组长度的n-1处。

    栈1为空时，top1等于-1；栈2为空时，top2=n。栈2是空栈，那么top1为n-1时就说明栈1满了；若栈1是空栈，top2为0时说明栈2满了。当top1+1=top2时，说明两个栈都满了。

    typedef struct
{
    ItemType data[MAXSIZE];
    int top1;
    int top2;
}MyDoubleStack;

Status Push (MyDoubleStack *S,ItemType e,int stackNum) {
    if(S->top1+1==S->top2) {
        return ERROR;                       //栈满
    }
    if (stackNum == 1) {
        S->data[++S->top1] = e;
    } else if (stackNum == 2) {
        S->data[--S->top2] = e;
    }
    return ok;
}

Status Pop (MyDoubleStack *S,ItemType *e,int stackNum) {
    if (stackNum == 1) {
        if(S->top1 == -1)  return ERROR;
        *e = S->data[S->top1--];
    } else if (stackNum == 2) {
        if(S->top1 == MAXSIZE)  return ERROR;
        *e = S->data[S->top2++];
    }
    return ok;
}

    栈的链式存储结构

    单链表有头指针，而栈顶指针也是必须的，因此将二者合二为一，将栈顶放在单链表的头部。有了栈顶指针，也就不需要链表的头结点了。

    对于空栈来说，链表原定义的头指针为空，那么链栈的空其实就是top=null。

    typedef Struct StackNode
{
    ItemType data；
    struct StackNode *next；
}StackNode,*LinkStackPtr;
typedef struct LinkStack
{
    LinkStackPptr top;
    int count;
}LinkStack;

    进栈

    Status Push (LinkStack *S, ItemType e) {
    LinkStackPtr s =  (LinkStackPtr) malloc (Sizeof(StackNode));
    s->data = e;
    s->next=S->top;                      //当前栈顶元素赋值给新节点的直接后继
    S->top = s;                            //将新节点的s赋值给栈顶指针
    S->count++；
    return ok;
 }

    出栈

     Status Pop (LinkStack *S, ItemType *e) {
    LinkStackPtr p;
    if (StackEmpty(%S)) return ERROR;
    *e = top->data;             
    p = S->top;                     //将当前的栈顶元素赋值给p
    S->top = S->top->next      //将栈顶指针指向栈顶的后一个节点
    free(p);
    S->count--;
    return ok;
 }

    链栈的存取操作没有涉及到循环，因此时间复杂度为O[1]。

    队列

    只允许在一端进行插入操作，在另一端进行删除操作的线性表。
    队列是一种先进先出的线性表，简称FIFO。允许插入的一端称为队尾，允许删除的一端称为队头。

    循环队列

    队列顺序存储的不足

    假设有一个n个元素的队列，需要建立一个大于n的数组，把队列的所有元素存储在数组的前n个单元，数组下标为0的一端即是队头。
    所有的入队操作就是在队尾追加一个元素，不需要移动任何元素的位置。因此时间复杂度为O[1]。

    由于队列的出列位置是在队头，下标为0的位置，也就意味着每取一个元素，后面所有的元素都要向前位移一个位置，因此时间复杂度为0[n]。

    如果考虑并不是每次从队列中取出元素都要移动其他元素，那么从队列中取出数据效率还是蛮高的。队列取出数据必须是从队头取，那么不从0下标的地方取，哪里是队头呢？我们定义一个front指针指向队列中的第一个元素，定义一个rear指针指向队尾元素的下一个位置，这样当front=rear时，即表示是一个空队列。

    上面的设计解决了队列取出数据要频繁移动元素的问题，但会引入新的问题：频繁的存入和取出元素，队列前面空余的位置越来越多，后面的位置原来越少，最终导致队列是有空间的，但存入数据却会数组越界。

    循环队列的定义

    头尾相接的顺序存储结构的队列被称为循环队列。
    接着上面的front指针和rear指针说事儿，头尾相接后终于用上了前面空出来的位置，但是当front=rear时，这个队列到底是空的还是满的呢？

    对于这个问题，我们修改队列满的条件，保留一个元素空间，也就是说一个队列最大尺寸是maxSize，当存储maxSize-1个元素时，就对外声称队列已经满了。此时判断队列是否满的条件是(rear+1)%maxSize == front。可能你会觉得用rear+1 == front不就行了，为啥还要那么费事儿，循环队列中头尾相接，谁也不知道rear那时候比front大还是小。

    循环队列的代码

    // 定义队列
typedef int QElemType;
typedef struct
{
    QElemType data[MAXSIZE];
    int front;
    int rear;
}MyQueue；
// 初始化一个空队列

{
    q->front =0;
    q->rear=0;
    return ok;
}

    return (Q.rear-Q.front+MAXSIZE)%MAXSIZE;
}
// 插入元素

{
    if((q->rear+1)%MAXSIZE == q->front) return ERROR; //对列已满
    q->data[q->rear] = e;
    q->rear=(q->rear+1)%MAXSIZE;

}
// 取出元素

{
    if(q->rear == q->front) return ERROR; //空队列
    *e = q->data[q->front];

    队列的链式存储

    队列的链式存储结构其实就是线性表的单链表，只不过它只能尾进头出。将队头指针指向链表的头结点，队尾指针指向终端结点。空队列时，front和rear指针都指向头结点。

    typedef int QElemType;
typedef struct QNode
{
    QElemType data;
    struct QNode *next;
}QNode,*QueuePtr;
typedef struct
{
    QueuePtr front,rear;
}LinkQueue;

    链式存储结构队列的入队操作

    入队操作其实就是在链表尾部插入节点。

    Status EnQueue (LinkQueue *Q,QelemType e)
{
    QueuePtr s = (QueuePtr) malloc (sizeof(Qnode));
    if (!s) exit(OVERFLOW);                     //分配内存失败
    s->ddata = e;
    s->next = NULL;
    Q->rear->next = s;
    return ok;
}

    链式队列的入队操作并没有涉及到循环，因此时间复杂度为O[1]。

    链式存储结构队列的出队操作

    出队操作就是头结点的后继节点出队，将头结点的后继改为出队节点后面的节点。若链表除头结点外只剩下一个元素时，则需将rear指向头结点。

    Status DeQueue (LinkQueue *Q, QelemType *e) 
{
    QueuePtr p;
    if(Q->front == Q->rear) return ERROR;  
    p = Q->front->next;
    * e = p->data;           
    Q->front->next=p->next
    if (Q->rear==p)
        Q->rear=Q->front;
   free(p);
   return ok;
}

    链式队列的出队操作并没有涉及到循环，因此时间复杂度为O[1]。

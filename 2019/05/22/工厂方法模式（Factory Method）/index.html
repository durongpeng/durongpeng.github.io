    工厂方法模式定义

    定义一个用于创建对象的接口，让子类决定实例化哪一个类。工厂方法使一个类的实例化延迟到其子类。

    工厂方法模式的例子

    场景：使用工厂方法模式创建卡片工厂制作卡片。

    Product类

    用来表示工厂生产的产品，是一个抽象类。

    public abstract class Product{
    public abstract void use();
}

    Factory类

    工厂类中使用了模板模式，该类还声明了用于”生产商品”的createProduct()抽象方法和用于”注册产品”的registerProduct抽象方法。两个抽象方法交给Factory的子类负责实现。
    抽象方法具体的实现内容根据工厂方法模式适用的场景不同而不同，只要是工厂方法模式，在生成实例时就一定会用到模板模式。

    public abstract class Factory{
    public final Product create(String owner){
        Product p = createProduct(owner);
    }
    public abstract Product createProduct(String owner);
    public abstract void registerProduct(Product product);
}

    IDCard类

    public class IDCard extends Product{
    private String owner;
    public IDCard(String owner){
        this.owner = owner;
    }
    // 实现Product抽象方法
    public void use(){
        System.out.println("use......");
    }
    public String getOwner(){
        return owner;
    }
}

    IDCardFactory类

    public class IDCardFactory extends Factory{
    private list owners = new ArrayList();
    // 实现Factory的抽象方法
    public Product createProduct(String owner){
        return new IDCard(owner);
    }
    // 实现Factory的抽象方法
    public void registerProduct(Product product){
        owners.add(((IDCard) prodcut).getOwner());
    }
    public List getOwners(){
        return owners;
    }
}

    Main调用

    public static void main(String[] args){
    Factory factory = new IDCardFactory();
    Product card1 = factory.crete("小明");
    Product card2 = factory.crete("小红");
}

    工厂方法模式的实现

    Product（产品）

    Product角色属于框架一方，是一个抽象类。它定义了在工厂方法模式中生成的那些实例所持有的接口(API)，但具体的实现由子类ConcreteProduct角色决定。

    Creator（创建者）

    Creator角色属于框架一方，它负责生成Product角色的抽象类，但具体的实现逻辑由其子类ConcreteCreator类来实现。
    Creator角色对于其子类ConcreteCreator如何生成Product是一无所知的，它只需要调用factoryMethod来生成Product即可。

    ConcreteProduct（具体的产品）

    ConcreteProduct角色属于具体加工的一方，它决定了具体的产品。

    ConcreteCreator（具体的创建者）

    ConcreteCreator橘色属于具体加工的一方，它负责生成具体的产品。

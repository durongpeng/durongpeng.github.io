    适配器模式定义

    将一个类的接口转换成客户希望看到的另外一个接口。Adapter模式使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。
    适配器模式分两种：

    • 类适配器模式(使用继承的适配器)
    • 对象适配器模式(使用委托的适配器)

      何时使用适配器模式

      让现有的类适配新的接口(API)时，常常会有”只要将这里稍微修改一下就可以了”的想法，一不留神就会产生bug。如果要对已经测试完毕的现有代码进行修改，就必须在修改后重新进行测试。使用Adapter模式可以在完全不改变现有代码的前提下使现有代码适配于新的接口(API)。

      类适配器模式

      类适配器例子

      场景：现有Banner类，两个方法：showWithParen()输出”(Hello)”，showWithAster()输出”#Hello#”。现想要复用Banner类输出点别的内容。
      示例程序图如下： 
      // Banner类
public class Banner{
  private String string;
  private Banner(String str){
      this.string = str;
  }
  public void showWithParen(){
      System.out.print("(Hello)");
  }
  public void showWithAster(){
      System.out.print("#Hello#");
  }
}
// Print接口
public interface Print{
  public abstract void printWeak();
  public abstract void printStrong();
}
// PrintBanner类
public class PrintBanner extends Banner implements Print{
  public PrintBanner(String str){
      super(str);
  }
  // 实现接口的方法
  public void printWeak(){
      // 调用Banner类的方法
      showWithParen();
      System.out.print("end");
  }
  // 实现接口的方法
  public void printStrong(){
      // 调用Banner类的方法
      showWithAster();
      System.out.print("end");
  }
}
// 调用main方法
public static void main(String[] args){
  Print p = new PrintBanner("Hello");
  p.printWeak();
  p.printStrong();
}
      调用者并不清楚PrintBanner是怎么完成的输出，PrintBanner作为适配器，调用了Banner的方法，这样复用了Banner类的代码，而且又没有对其进行修改，不影响现有的程序。

      对象适配器模式

      场景：上述类适配器场景中，PrintBanner类实现Print接口，继承Banner类。但若是Print不是一个接口而是一个抽象类，Java又不支持多继承，那就不好办了。于是我们让PrintBanner继承Print抽象类，并且持有一个Banner类的对象。
      // Print抽象类
public abstract class Print{
  public abstract void printWeak();
  public abstract void printStrong();
}
// PrintBanner
public class PrintBanner extends Print{
  private Banner banner;
  public PrintBanner(String str){
      this.banner = new Banner(str);
  }
  public void printWeak(){
      banner.showWithParen();
  }
  public void printStrong(){
      banner.showWithAster();
  }
}

      适配器模式的实现

      适配器模式中的角色

      Target（对象）

      该角色负责定义所需要的方法。

      Client（请求者）

      该角色负责使用Target角色所定义的方法进行具体处理。

      Adaptee（被适配）

      Adaptee是一个持有既定方法的角色，例子中Banner类扮演该角色。

      Adapter（适配器）

      使用Adaptee角色的方法来满足Target角色的需求，这是Adapter模式的目的，也是Adapter角色的作用。

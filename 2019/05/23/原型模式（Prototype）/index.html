    原型模式定义

    用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象。

    使用原型模式的情况

    • 1.对象种类繁多，无法将它们整合到一个类中。
      第一种情况是需要处理的对象太多，如果将它们分别作为一个类，就必须要编写很多个类文件。
    • 2.难以根据类生成实例时。
      第二种情况是生成实例的过程太过复杂，很难根据类来生成实例。例如一个类有N多个属性，new出来一个空对象，一个一个setter方法为这些属性赋值，真的太麻烦了。
      通常在想生成一个和之前用户通过操作创建出的实例完全一样的实例时，我们会事先将用户通过操作生成的实例保存起来，然后再需要时通过复制来生成新的实例。
    • 3.想解耦框架与生成的实例时。
      第三种情况是想要让生成实例的框架不依赖于具体的类。这时，不能指定类名来生成实例，而要事先”注册”一个”原型”实例，然后通过复制该实例来生成新的实例。

      原型模式例子

      Product接口

      Product是提供复制功能的接口。该接口继承了java.lang.Cloneable接口。
      public interface Product extends CloneAble{
  public abstract void use(String s);
  public abstract Product createClone();
}

      Manager类

      Manager类使用Product接口来复制实例。
      public class Manager{
  private HashMap showcase = new HashMap();
  public void register(String name,Product p){
      showcase.put(name,p);
  }
  public Product create(String protoname){
      Product p = (Product) showcase.get(protoname);
      return p.createClone();
  }
}
      Manager类中并没有出现Product实现类的名字(MessageBox和UnderLinePen)，这意味着我们可以独立修改Manager和Product，不受Product实现类的影响。这是非常重要的，因为一旦在类中使用到了别的类名，就意味着该类和其他类紧密耦合在一起。在这个例子中，Product成了Manager类和其他类连接的桥梁。

      MessageBox类和UnderLinePen类

      都是Product接口的实现类。decochar字段中保存的是装饰框字符，use方法是用decochar字段中保存的字符将要显示的文本装饰起来。createClone方法用于复制自己，内部调用clone方法，我们之所以可以调用clone方法进行复制是因为实现了java.lang.Cloneable接口。
      需要注意的是，java.lang.Cloneable接口只是起到告诉程序可以调用clone方法的作用，它自身并没有定义任何方法。clone方法是继承于Obejct的。

      Main方法

      public static void main(String[] args){
  Manager manager = new Manager();
  Product upen = new UnderLinePen("~");
  Product mbox = new MessageBox("*");
  manager.register("upen",upen);
  manager.register("mbox",mbox);
  // 生成Product
  Product uProduct = manager.create("upen");
  uProduct.user("Hello");
  Product mProduct = manager.create("mbox");
  mProduct.user("Hello");
}

      原型模式实现

      Prototype（原型）

      Prototype角色负责定义用于复制现有实例来生成新实例的方法。在上例中，Product接口负责扮演此角色。

      ConcretePrototype（具体的原型）

      ConcretePrototype角色负责实现复制现有实例并生成新的实例的方法。在上例中，MessageBox和UnderLinePen负责扮演此角色。

      Client（使用者）

      Client角色负责使用复制实例的方法生成新的实例。在上例中，Manager类扮演此角色。
